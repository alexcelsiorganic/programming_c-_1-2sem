! package не использовать

Во всех входных файлах каждая запись находится на новой строке.
Поля одной записи отделяются друг от друга символом точка с запятой.
В конце строки точка с запятой НЕ ставится.
!!! В выходных файлах не должно быть дополнительных пустых строк в конце.
Примеры входных и выходных файлов вы можете посмотреть на вкладке Tests / Тест кейсы для данной задачи.


Во входном файле input1.txt находится информация о вакансии. Каждая вакансия описана с новой строки.
Приоритет для поиска вакансии priority - А (max) В С D (min)



Во входном файле input2.txt находится информация о рекрутерах. Информация о каждом рекрутере с новой строки.
recruter_id;recruter_surname;recruter_name


Во входном файле input3.txt описаны правила начисления бонусов для рекрутера за закрытие вакансии. Каждое правила с новой строки: приоритет вакансии (A B C D), количество дней <= days, за которые рекрутер закрыл вакансию и сумма бонуса. Если количество дней, за которые была закрыта вакансия, превышает максимальное число дней в правилах, то бонус равен 0. Например,
А;15;100
A;30;75
A;45;50
B;15;70 и тд
priority;days;bonus


Во входном файле input4.txt находится список закрытытия вакансий, каждая запись с новой строки: id вакансии, дата открытия и дата закрытия вакансии, id рекрутера, который закрыл вакансию. Если вакансия не закрыта, то data_close = 0. Формат даты: dd.mm.yyyy



2 балла. В выходной файл output1.txt вывести все вакансии по убыванию приоритета (от A до D). Каждая вакансия с новой строки. Для вакансий с одинаковыми приоритетами вывести в порядке следования во входном файле input1.txt.
vacancy_id;title;priority


2 балла. В выходной файл output2.txt вывести список просроченных вакансий в порядке убывания id вакансии. Вакансию будем называть просроченной, если она не закрыта (data_close = 0) и от даты открытия вакансии до текущей даты (сегодня) прошло >= 90 дней.
vacancy_id;data_open


2 балла. В выходной файл output3.txt вывести фамилию и имя всех рекрутеров (каждый рекрутер 1 раз, каждый рекрутер с новой строки), у которых есть хотя бы 1 незакрытая вакансия, по возрастанию id рекрутера.
recruter_surname;recruter_name


4 балла. В выходной файл output4.txt вывести по заданному recruter_id (входной файл input5.txt) сумму бонуса за все закрытые вакансии. Если рекрутера с таким id нет, то вывести 0.
